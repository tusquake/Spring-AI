# Topic 41: AI Observability & Monitoring (Micrometer)

## Overview
When deploying Spring AI applications to production, "it works on my machine" is not enough. You must answer critical questions:
- *How much money did that prompt cost?*
- *Why did the LLM take 15 seconds to reply?*
- *How many tokens were generated by the HelpDesk agent today?*

Spring AI provides auto-configured Observability through **Micrometer**, allowing you to trace requests across microservices and export metric data seamlessly to Prometheus, Datadog, Zipkin, or Grafana.

## 🛠️ Configuration Setup

### 1. Add Dependencies
Include the Spring Boot Actuator and the Micrometer tracing backend of your choice.

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
<dependency>
    <!-- Tracing context propagation -->
    <groupId>io.micrometer</groupId>
    <artifactId>micrometer-tracing-bridge-brave</artifactId>
</dependency>
```

### 2. Enable in Application Properties
You must explicitly turn on observation for Spring AI models.

```yaml
spring:
  ai:
    chat:
      client:
        observations:
          enabled: true
management:
  tracing:
    sampling:
      probability: 1.0 # Sample 100% of requests for testing
  endpoints:
    web:
      exposure:
        include: "*"
```

## 🧠 What gets Monitored?

By simply enabling these properties, Spring AI wraps every `ChatClient`, `VectorStore`, and `EmbeddingModel` call with a Micrometer `Observation`.

It automatically records:
1. **Request Latency**: Exactly how long the AI provider (OpenAI/Google) took to respond.
2. **Tokens Configured vs Used**: How many Prompt and Completion tokens were burned.
3. **Model Specifics**: Which exact model version was invoked (e.g., `gpt-4o-mini`).
4. **Error Rates**: Capturing `429 Too Many Requests` or `5xx` gateway timeouts natively.

### Distributed Tracing Example
Because it integrates with Micrometer, an incoming HTTP Request ID (`trace-id: 1234a`) is automatically propagated down into the Spring AI HTTP client. If a user complains "my ticket creation timed out," you can look up `1234a` in Jaeger/Zipkin and see exactly what the LLM was doing at that precise moment.

## Summary
Never fly blind in production. Enabling Spring AI Observations takes zero custom Java code but provides enterprise-grade metrics essential for billing, cost optimization, and SLA monitoring.
